# 실험 진행 가이드 (실험자용)

## 사전 준비

1. **URL 안내**: 카톡으로 참가자에게 배포된 Vercel URL 전송
   예: `https://ergonomics-experiment.vercel.app`
2. **수동 시간 측정 도구**: 스톱워치 or 폰 타이머. 퀴즈 풀이 시작/끝 시점 측정용.
3. **상호작용 통화/화면 공유**: 가능하다면 줌으로 화면 공유 받기 → 인터럽션 발동/복귀 관찰

## 진행 절차

### 0. 시작 전 안내
- "실험은 약 20-25분, 두 종류의 UI로 동일 주제의 응답을 읽고 퀴즈를 풉니다"
- "도중에 잠깐 다른 화면이 뜰 수 있는데 그건 의도된 인터럽션입니다"
- "퀴즈는 정답이 중요하지 않으니 본인이 이해한 대로 적어주세요"

### 1. 참가자가 URL 접속 → 동의 체크 → ID 확인
- 자동 부여된 ID 그대로 두어도 되고, 본인이 알아볼 수 있는 닉네임으로 바꿔도 됨
- 참가자 ID를 별도 노트에 기록

### 2. 사전 설문 → 연습 페이지
- 청크 한 번 펼치고/접고, 북마크도 눌러보게 하기
- "북마크는 사이드바에 표시됩니다" 안내

### 3. 조건 1 시작
- "지금부터는 데이터가 기록됩니다. 천천히 읽으세요"
- **퀴즈 시작 시점에 스톱워치 시작, 제출 시점에 정지** → 초 단위 기록

### 4. NASA-TLX
- "방금 읽으신 응답과 퀴즈 과제 전반에 대한 주관적 부담을 평가합니다"
- 0=매우 낮음, 100=매우 높음

### 5. 휴식 → 조건 2 (반대 UI)
- 같은 절차 반복. 인터럽션은 다른 UI에서 발동됨

### 6. 마무리 인터뷰 → 자동 제출
- 제출이 성공하면 ✓ 표시
- 만약 실패 메시지가 뜨면 "JSON 백업 받기" 클릭해서 파일 받아오기

## 사후 작업

1. Google Sheet의 `responses` 탭 확인 → 참가자 행이 들어왔는지
2. `q1_quiz_time_manual_s`, `q2_quiz_time_manual_s` 컬럼에 수동 측정한 퀴즈 시간(초) 입력
3. `q1_quiz_ans1/ans2`, `q2_quiz_ans1/ans2` 채점 (별도 채점 시트로 복사하거나 옆 컬럼에 점수)
4. 정성 코멘트는 별도 시트에 정리

## 트러블슈팅

- **모달이 안 뜸**: 트리거 조건 미달성. URL에 `?interrupt=time45` 같이 강제 지정 가능.
- **시간이 이상하게 큰 값**: 참가자가 탭 전환했을 가능성. visibility change 로그는 별도 기록 안 됨 → 본인 노트에 메모.
- **제출 실패**: JSON 백업 받아두기 + Apps Script 권한 재확인.
